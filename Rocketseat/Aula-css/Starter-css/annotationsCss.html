O que significa CSS?

  • Cascading Stryle Sheet
  • Código para criar estilos no html
  • Html é a estrutura, e o css é a beleza
  • É uma linguagem style sheet

Anatomia

'nome da tag' {
      propriedade: valor;
      color: blue;
}
  • Selector : prcoura por todos os 'nome da tag'
  • Declaretion : O que esta dentro das chaves, que vai ser aplicadas aos 'nome da tag' que forem encontradas no html
  • Properties : propriedade
  • Property Value : valor da propriedade

Selectors

Conecta um elemento html com o css

Tipos
  • Global selector (*)
  • Element/type selector (h1, h2, p, div)
  • ID Selector (#nomeDoIDnohtml)
  • Class Selector (.nome)
  • Attribute selector, Pseudo-class, Pseudo-element, etc

HTML

<div id="nomeDoID" class="nomeClass">
      <h1>Título</h1>
</div>

CSS

#nomeDoID {
      width: 200px;
}

.nomeClass {
      margin: 40px; }

h1 {
      color:pink;
      font-size: 40px;
      background: black;
}

Box model (caixas)

  • Quase tudo são caixas do css
  • Posicionamentos, tamanhos, espaçamentos, bordas, cores
  • Caixa pode ficar do lado de outra, acima ou abaixo
  • Elementos html são caixas

Adicionando CSS

1. inline
atributo 'style', dentro da tag (style="")
2. <style>
tag html que irá conter o css, dentro do corpo html(head)
3. <link>
arquivo css externo
4. @import
arquivo css interno

A cascata

Cascading

A ordem de imporância é de cima para baixo, também funciona com a tag style no html.

Levando em consideração 3 fatores:
  1. Origem do estilo
  2. Especificidade
  3. Importância

1. Origem do estilo

inline > tag style > tag link

2. Especificidade

Cálculo matemático, onde, cada tipo de seletor e origem do estilo, possuem valores a serem considerados

Força Tipo de seletor e origem do estilo
0 Universal selector(*), combinators e negation pseudo-class(:not())
1 Element type selector e pseudo-elements(::before, ::after)
10 Classes e attribute selector([type="radio"])
100 ID Selector
1000 Inline

Regra !important

  • Não é considerado boa prática
  • Quebra o fluxo natural da cascata
  • Ela sobrepõe tudo (!important > inline)

At-rules

  • Relacionado ao comportamento css
  • Começa com o sinal @ seguido do identificador e valor

Exemplos

At-rules Descrição No css
@import Incluir um css externo @import url ("http://local.com/style.css");
@media Regras condicionais para dispositivos @media (min-width: 500px) {
rules here
}
@font-face Fontes externas @font-face {
rules here
}
@keiframes Animation @keyframes nameofanimation {
rules here
}

Shorthand

  • Junção de propriedades
  • Resumido
  • Legível
  • Não considera propriedades anteriores
  • Valores não especificados assumem o valor padrão
  • geralmente a ordem descrita não importa
  • Se ouver muitas propriedades com valores semelhantes pode ocorrer um problema
  • Algumas propriedades não aceitam shorthand

Exemplo: background

Background Properties:

background-color: #000;
background-image: url(images/bg.gif);
background-repeat: no-repeat;
background-position: left top;

Background Shorthand:

background: #000 url(images/bg.gif) no-repeat left top;

Funções

  • Nome seguido de parentesis
  • Recebe argumentos

Exemplo

@import url ("http://urlaqui.com/style.css");
{
color: rgb(255, 0, 100);
width: calc(100% - 10px);
}

Cuidado com a escrita

Identação é muito importante:

  • Prestar muita atenção nos espaços, pontos e vírgulas, etc.

Exemplo

Modo correto:

p {
      margin: 0 auto;
      padding-left: 15px;
}

Vendor Prefixes

(Prefixo do browser)

Permite que browsers adicione 'features' a fim de colocar em uso alguma novidde que vemos no css

Exemplo

p {
      -webkit-background-clip: text;         (Chrome, Safari, IOS, Android)
      -moz-background-clip: text;            (Mozilla (firefox))
      -ms-background-clip: text;              (Internet Explorer)
      -o-background-clip: text;                (Opera)
}
